ALP 1.0

These are ALP 1.0 release notes.

Changes affecting all architectures

Information in this section applies to all architectures supported by ALP 1.0.

Basic utilities

Remove targetpw mode in sudoers

The default sudoers/root configuration was confusing due to it duplicating the first user’s password for the root account and relied on a unique sudoers setting (Defaults targetpw) that asked for the root’s password instead of the user’s for sudo commands. This approach was uncommon and surprising compared to other distributions that either did not set a root password or set a separate one and used the user’s password for sudo, aligning with most users' expectations.

This has been changed and by default targetpw is not used.

Miscellaneous

systemd uses cgroup v2 by default

ALP 1.0 uses cgroup v2 by default and v1 is unsupported. If you need to use cgroup v1, SLES 15 SP6 can be switched to hybrid mode using a boot parameter.
